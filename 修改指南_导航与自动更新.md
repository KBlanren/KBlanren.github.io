# 导航栏与自动更新 —— 逐行修改教程

> 目标：
> 1. BLOG 从「下拉菜单」变成「普通链接」，点击后显示博客文章列表页；
> 2. 新增 HORROR 下拉菜单，从 `content/horror/` 文件夹自动读取 Markdown 文件；
> 3. 以后新增文章后，运行 `build.py` 即可自动更新导航，无需手动改 HTML。

---

## 先理解一个限制：为什么需要 `build.py`

浏览器里的 JavaScript **不能直接读取服务器上的文件夹列表**（安全策略限制）。
所以「网页自动更新下拉栏」的真实做法是：

- 你新建 `content/horror/某篇小说.md`
- 运行一次 `python build.py`
- 脚本自动扫描所有目录，生成 `content/manifest.json`
- 网页打开时读取这个 `manifest.json`，动态渲染下拉菜单

> 每次新增、删除、重命名文章后，**双击运行 `build.py` 即可**，不用改任何代码。

### 3.2 新增 manifest 读取 + 动态渲染（第 39–124 行）

在 `updateHeader` 函数**之前**插入了三大块代码：

#### ① `loadManifest()` —— 读取文章清单

```js
let contentManifest = null;

async function loadManifest() {
    try {
        const resp = await fetch('content/manifest.json');   // 读取 build.py 生成的清单
        if (!resp.ok) throw new Error('HTTP ' + resp.status);
        contentManifest = await resp.json();               // 解析成 JS 对象
    } catch (err) {
        console.error('Manifest load failed:', err);
        contentManifest = {};                                // 如果读取失败，设为空对象，防止报错
    }
}
```

> 作用：网页打开时，先加载 `content/manifest.json`，拿到所有分类的文章列表。这个文件是「桥梁」——前端靠它知道有哪些文章、标题叫什么、路径是什么。

---

#### ② `renderDropdowns()` —— 动态渲染 HORROR 下拉菜单

```js
function renderDropdowns() {
    if (!contentManifest) return;

    // 桌面端：往 id="horrorDropdown" 的 div 里塞链接
    const horrorDesktop = document.getElementById('horrorDropdown');
    if (horrorDesktop && contentManifest.horror) {
        horrorDesktop.innerHTML = contentManifest.horror.map(item =>
            `<a href="#content" class="dropdown-item" data-path="${item.path}">${item.title}</a>`
        ).join('');
    }

    // 移动端：往 id="mobileHorrorDropdown" 的 div 里塞链接
    const horrorMobile = document.getElementById('mobileHorrorDropdown');
    if (horrorMobile && contentManifest.horror) {
        horrorMobile.innerHTML = contentManifest.horror.map(item =>
            `<a href="#content" class="mobile-dropdown-item" data-path="${item.path}">${item.title}</a>`
        ).join('');
    }

    // 关键：新插入的链接必须重新绑定点击事件，否则点了没反应
    document.querySelectorAll('.dropdown-item[data-path], .mobile-dropdown-item[data-path]').forEach(link => {
        link.removeEventListener('click', handleNavClick);  // 先删，防止重复绑定
        link.addEventListener('click', handleNavClick);   // 再绑
    });
}
```

> 逐行解释：
> - `contentManifest.horror` 是一个数组，每个元素是 `{file: "xxx.md", title: "xxx", path: "horror/xxx"}`
> - `.map(...).join('')` 把数组转换成一串 HTML 字符串，一次性插入到页面里
> - 最后必须重新绑定 `handleNavClick`，因为这些链接是 JS 动态生成的，初始 HTML 里没有它们，所以 `DOMContentLoaded` 时的绑定轮不到它们

---

#### ③ `renderCategoryList(category)` —— 渲染 BLOG 这样的「分类列表页」

```js
function renderCategoryList(category) {
    const items = contentManifest[category] || [];   // 从 manifest 取该分类的所有文章

    // 如果该分类一篇都没有，显示「暂无内容」提示
    if (items.length === 0) {
        contentArea.innerHTML = `...`;
        return;
    }

    // 每篇文章渲染成一个方块链接
    const listHtml = items.map(item => `
        <a href="#${item.path}" class="category-list-item" data-path="${item.path}">
            <span class="category-list-title">${item.title}</span>
            <span class="category-list-arrow">→</span>
        </a>
    `).join('');

    // 整体组装成列表页
    contentArea.innerHTML = `
        <div class="category-list">
            <h1>${category.toUpperCase()}</h1>
            <div class="category-list-items">${listHtml}</div>
        </div>
    `;

    // 绑定点击：点了某篇文章后，跳转到对应 path 并加载 Markdown
    document.querySelectorAll('.category-list-item').forEach(link => {
        link.addEventListener('click', (e) => {
            e.preventDefault();
            const path = link.dataset.path;
            if (path) {
                window.location.hash = path;
                closeMobileMenu();
            }
        });
    });

    // 触发 MathJax 渲染（如果标题里有公式）
    if (window.MathJax) {
        MathJax.typesetPromise([contentArea]).catch(() => {});
    }
}
```

> 作用：BLOG 不再是下拉菜单，点击后进入「文章列表页」，列出所有博客文章。每篇文章是一个可点击的方块，点击后加载对应 Markdown。这是「非下拉分类」的通用渲染函数，以后 PAPERS、COURSES 等分类也可以复用。

### 3.3 改 handleRoute()（第 208–226 行）

**修改前：**
```js
function handleRoute(path) {
    ...
    if (path === 'home' || !path) {
        loadHome();
    } else {
        loadMarkdown(path);
    }
}
```

**修改后：**
```js
function handleRoute(path) {
    ...
    if (path === 'home' || !path) {
        loadHome();
    } else if (path === 'blog') {       // 新增：判断是不是 BLOG 分类入口
        updateHeader('blog');
        renderCategoryList('blog');       // 渲染博客文章列表页
    } else {
        loadMarkdown(path);               // 其他路径（单篇文章）直接加载 Markdown
    }
}
```

> 解释：当 URL hash 是 `blog` 时，不加载某个具体 Markdown，而是调用 `renderCategoryList('blog')` 显示列表页。当 hash 是 `blog/2026陆家嘴论坛总结` 这种单篇文章路径时，走 `else` 加载具体 Markdown。

---

### 3.4 改 DOMContentLoaded（第 297–300 行）

**修改前：**
```js
document.addEventListener('DOMContentLoaded', () => {
    // 直接绑定事件...
});
```

**修改后：**
```js
document.addEventListener('DOMContentLoaded', async () => {
    await loadManifest();      // 先等 manifest 加载完
    renderDropdowns();         // 再渲染下拉菜单

    // 然后绑定事件（不变）...
});
```

> 关键：加了 `async`，并且把 `loadManifest()` 和 `renderDropdowns()` 放在最前面。如果顺序颠倒，下拉菜单会在 manifest 还没读到时渲染，结果就是空的。

## 五、build.py 是什么（怎么用）

这是一个 **Python 脚本**，作用是扫描你放在 `content/` 下的文件夹，自动生成 `content/manifest.json`。

### 文件内容核心逻辑

```python
CATEGORIES = {
    "blog":  {"type": "list",    "label": "Blog"},     # 只生成列表页，不生成下拉
    "horror": {"type": "dropdown", "label": "Horror"},  # 生成下拉菜单
    "papers": {"type": "dropdown", "label": "Papers"}, # 同理，可扩展
    "courses": {"type": "dropdown", "label": "Courses"},
    "dsge": {"type": "dropdown", "label": "DSGE"},
}
```

> 你想新增一个分类（比如 `essays`），就在这个字典里加一行：`"essays": {"type": "dropdown", "label": "Essays"}`，然后在 `content/` 下新建 `essays/` 文件夹即可。

### 使用方式

1. 在 `content/horror/` 下新建 Markdown 文件，比如 `午夜电梯.md`
2. 打开终端（或命令行），进入项目文件夹
3. 运行：
   ```bash
   python build.py
   ```
4. 脚本会自动生成 `content/manifest.json`，里面包含 horror 的新文章
5. 刷新网页，HORROR 下拉菜单里就会出现「午夜电梯」

> **注意**：你的环境可能需要 `python3` 或 `py` 命令，如果 `python` 报错就试试 `python3 build.py` 或 `py build.py`。

## 六、content/manifest.json 长什么样

这是 `build.py` 生成的一个示例（目前只有一篇博客）：

```json
{
  "blog": [
    {
      "file": "2026陆家嘴论坛总结.md",
      "title": "2026陆家嘴论坛总结",
      "path": "blog/2026陆家嘴论坛总结"
    }
  ]
}
```

以后变成这样：
```json
{
  "blog": [
    { "file": "2026陆家嘴论坛总结.md", "title": "2026陆家嘴论坛总结", "path": "blog/2026陆家嘴论坛总结" }
  ],
  "horror": [
    { "file": "午夜电梯.md", "title": "午夜电梯", "path": "horror/午夜电梯" },
    { "file": "阁楼里的脚步声.md", "title": "阁楼里的脚步声", "path": "horror/阁楼里的脚步声" }
  ]
}
```

> 你**不需要手动编辑**这个文件，运行 `build.py` 即可自动生成。但如果你在没有 Python 的环境（比如纯 GitHub Pages 发布），可以手动维护这个 JSON，格式保持一样就行。

## 八、常见问题

**Q：运行 `build.py` 报错找不到 python？**
> 试试 `python3 build.py` 或 `py build.py`。如果都没有，就手动编辑 `content/manifest.json`，按格式添加文章即可。

**Q：HORROR 下拉菜单里没出现文章？**
> 检查：① `content/horror/` 下是否有 `.md` 文件；② 是否运行了 `build.py` 生成 `manifest.json`；③ 浏览器控制台（F12 → Console）有没有报错。

**Q：BLOG 点击后显示「暂无内容」？**
> 说明 `manifest.json` 里没有 `blog` 数组，或者数组为空。运行 `build.py` 即可。

**Q：想给 HORROR 下的文章排序？**
> `build.py` 里默认按文件名排序（`sorted`）。如果你想自定义顺序，可以给文件名加前缀数字，比如 `01_午夜电梯.md`、`02_阁楼里的脚步声.md`，排序就会按数字来。
